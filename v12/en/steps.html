Using a Throw node to throw an exception in a message flow

IBM Knowledge Center
Learn how to use the throw node to throw an exception and use the Compute node to navigate the resulting ExceptionList.

Import projects

When you click Import, one project will be created in your workspace: An application named ExampleThrowException.

ExampleThrowException provides a single message flow named ThrowException which receives data using an HTTPInput node. The next node in the flow is a Throw node which generates a user exception. The message is returned to the HTTPInput node and sent down its Catch terminal. Next the Compute node ESQL navigates through the ExceptionList structure, extracts the human readable text explaining the reason for the exception and embeds it within a suitable reply which is returned using the HTTPReply node at the end of the flow.

Once imported using the link on the right, the provided project is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Create an integration node and deploy the message flow

Prepare the scenario by executing the following steps:

  1. Create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  2. Back in the Toolkit, drag and drop deploy the provided BAR file named ThrowException.bar which you will find inside the ExampleThrowException application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open ThrowException.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Click the Send Message icon .
    • A saved message is provided.
      The input message InputMessage contains this data:
      {"Message": "This can be anything"}
      Select the input message and click Send and your message is sent to the HTTPInput node.
    • After you close the dialog, the path taken through the messageflow is highlighted.

This tutorial has shown how to use the Throw node to throw an exception and how to use the Compute node to navigate the resulting ExceptionList.
